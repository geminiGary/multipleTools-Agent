# RAG 验证清单

本文档用于验证本地知识库检索是否按预期工作。项目内置的测试语料是虚构产品“知简笔记”，位于 `data/docs/`。这些事实不依赖模型预训练知识，因此非常适合验证 RAG 是否真正生效。

## 启动前检查

1. `.env` 中已配置可用的聊天模型。
2. `data/docs/` 下存在 `.md` 或 `.txt` 文档。
3. 启动服务时终端输出了类似 `已从 data/docs 入库 N 个文本块` 的信息。
4. 打开 `GET /api/capabilities`，确认返回中 `rag_enabled` 为 `true`。
5. Web 端右侧能力栏显示 RAG 已启用。

## 域内问题

以下问题应该能从知识库中找到答案，并在右侧仪表栏显示引用来源。

| # | 问题 | 预期答案 | 预期出处 |
|---|---|---|---|
| 1 | 知简专业版多少钱？ | ¥18/月 或 ¥168/年 | 知简-价格与套餐 |
| 2 | 免费版最多能存多少篇笔记？ | 100 篇 | 知简-价格与套餐 |
| 3 | 全局搜索的快捷键是什么？ | Ctrl+Shift+F | 知简-功能与快捷键 |
| 4 | 团队版的版本历史保留多久？ | 1 年 | 知简-价格与套餐 |
| 5 | 知简能导入哪些来源的笔记？ | Evernote（.enex）和 Notion 导出包 | 知简-功能与快捷键 |
| 6 | 知简的退款政策是怎样的？ | 7 天无理由退款 | 知简-客服与隐私 |
| 7 | 免费版最多能在几台设备上同步？ | 5 台 | 知简-价格与套餐 / 知简-客服与隐私 |
| 8 | 知简的数据会用于训练 AI 模型吗？存在哪里？ | 不会用于训练；存于“华东节点”（上海） | 知简-客服与隐私 |
| 9 | 知简当前最新版本号是多少？ | v4.2（2026-03） | 知简-产品概览 |

## 域外问题

以下问题不应被模型编造。系统应明确说明知识库未提及，或交给更合适的工具处理。

| # | 问题 | 期望行为 |
|---|---|---|
| A | 知简笔记的创始人/CEO 是谁？ | 说明知识库未提及，不编造人名 |
| B | 知简和印象笔记哪个更好？ | 说明没有对比依据，避免主观扩写 |
| C | 北京今天天气怎么样？ | 与知识库无关，应由天气工具处理 |

## 调试建议

- 如果 `rag_enabled=false`，先确认 `data/docs/` 是否有文档，并重启 `uvicorn`。
- 如果引用来源为空，检查 `KnowledgeBaseTool.last_sources` 是否在命中结果后写入。
- 如果答案与文档不一致，检查 Agent 是否使用了预检索结果，以及系统提示是否要求只依据知识库回答。
- 如果检索命中不稳定，可以调整 `top_k`、chunk 大小、重叠长度或 rerank 规则。
- 如果 embedding 服务不可用，确认 `.env` 中 `LOCAL_EMBEDDING_FALLBACK=true`，让本地哈希向量兜底生效。
