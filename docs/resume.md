# 简历项目包装

这份文档用于把“智能多工具助手”整理成简历条目和面试讲解材料。

## 一句话概括

基于 OpenAI 兼容 API 设计并实现一个多工具智能体系统，支持 Function Calling、SSE 流式响应、本地 RAG 知识库、多会话持久化和跨会话长期记忆。

## 简历项目描述

**智能多工具助手 | Python / FastAPI / OpenAI API / RAG / Function Calling**

- 设计并实现多工具智能体主循环，支持模型流式输出、function calling 工具调用、工具结果回填和多轮工具调用上限控制。
- 基于 FastAPI 和 SSE 构建后端对话接口，实现 Web 端 token 级流式响应、工具调用过程展示和引用来源展示。
- 抽象 `Tool` 与 `ToolRegistry`，统一管理天气、计算器、网络搜索和本地知识库工具，支持工具可用性判断与异常兜底。
- 实现本地 RAG 知识库能力，支持文档切块、embedding 入库、余弦相似度检索、来源追踪和“资料未提及”防幻觉策略。
- 实现短期记忆压缩和长期记忆持久化，支持跨会话召回用户事实、偏好更新和用户主动删除记忆。
- 实现多会话管理和本地持久化，支持会话创建、切换、重命名、删除和服务重启后的上下文恢复。
- 使用 Pytest 覆盖 Agent 事件流、工具注册、RAG、长期记忆、会话持久化和后端 API，当前本地测试 `60 passed`。

## 面试讲解顺序

1. **为什么做这个项目**：普通聊天机器人只能回答文本，缺少外部工具、私有知识库和长期记忆，所以做了一个可扩展的 Agent 原型。
2. **核心架构**：前端/CLI 进入 FastAPI，后端创建 Agent，Agent 统一调度 LLM、工具注册表、记忆系统和 RAG 检索。
3. **工具调用链路**：模型只负责选择工具和参数，真实执行由 `ToolRegistry.dispatch()` 完成，结果再回填给模型生成最终回答。
4. **RAG 怎么做**：启动时把 `data/docs` 文档切块并向量化，提问时检索 top-k 片段，把结果和来源注入模型上下文。
5. **记忆怎么做**：短期记忆保留近期原文并压缩旧历史，长期记忆抽取稳定事实落盘，再按当前问题做向量召回和关键词兜底。
6. **工程化处理**：工具异常不打断对话，搜索和 embedding 都有兜底方案，会话和记忆都持久化，并用测试覆盖关键路径。

## 可继续增强的点

- 使用持久化向量数据库替换内存向量库。
- 为 RAG 增加 rerank、置信度阈值和更细粒度引用。
- 增加真实天气、日历、邮件、数据库查询等外部工具。
- 增加登录鉴权和多用户隔离。
- 将前端拆分为独立工程并引入组件化框架。
